Wooden Ships Database API Landing Page

All resources will return a JSON object that contains the following fields:

  • timestamp:The date of the query.
  • query:The sql text of the query (for debugging).
  • success:Boolean showing whether the query succeeded or failed at the database level.
  • Data:An array of objects. Object details are outlined below.

All parameters and URIs are CaSe SeNSitiVe. Use caution.

Available Resources:

  • Captains.php Returns a list of captains and their ranks and vessels in JSON format.
    • Accepts Parameters:
      • shipName: Exact name of ship.
      • shipType: Exact name of type of ship.
      • nationality: Exact name of nationality.
      • captainName: Exact name of captain.
      • rank: Exact rank name.
      • minDate: Time filter specifying the earliest activity of this captain. Format: yyyy-mm-dd
      • maxDate: Time filter specifying the latest voyage start date of this captain. Format: yyyy-mm-dd
    • Returns Fields:
      • Captain: Name of captain.
      • CaptainRank: Rank of captain.
      • shipName: Name of ship Sailed.
      • Company: Name of associated company.
      • Nationality: Name of assoicated nationality.
    • Examples:
      • Return all the British Captains.
      • Return all the captains sailing ships of type 'Sloop'.
      • Return all the captains operating between 1800 and 1805.
  • Companies.php Returns a list of Companies and their nationalities in JSON format.
    • Accepts Parameters:
      • nationality: Exact name of nationality assoicated with these companies.
      • minDate: Time filter specifying the earliest activity of this company. Format: yyyy-mm-dd
      • maxDate: Time filter specifying the latest voyage start date of this company. Format: yyyy-mm-dd
    • Returns Fields:
      • Company: Name of associated company.
      • Nationality: Name of assoicated nationality.
      • companyID: Database ID for this company entity
    • Examples:
      • Return all the Dutch companies.
      • Return all the captains operating between 1750 and 1800.
  • data.php Returns logbook details for individual records. Allows flexible querying of the weather parameters and can include the text observations in the respose. Returns in JSON format.
    • Accepts Parameters:
      • shipName: Exact name of ship.
      • shipType: Exact name of type of ship.
      • nationality: Exact name of nationality.
      • captainName: Exact name of captain.
      • minDate: Time filter specifying the minimum for the temporal search window. Format: yyyy-mm-dd
      • maxDate: Time filter specifying the maximum for the temporal search window. Format: yyyy-mm-dd
      • minAirTemp: Minimum air temperature in degrees celsius.
      • maxAirTemp: Maximum air temperature in degrees celsius.
      • minPressure: Minimum air pressure in degrees millimeters mercury.
      • maxPressure: Maximum air pressure in degrees millimeters mercury.
      • gusts: Was it gusty?
      • rain: Was rain recorded?
      • snow: Was snow recorded?
      • hail: Was hail recorded?
      • thunder: Was thunder recorded?
      • fog: Was it foggy?
      • seaice: Was sea ice encountered?
      • windSpeedMin: Minimum wind speed in meters per second.
      • windSpeedMax: Maximum wind speed in meters per second.
      • windDirMin: Minimum wind direction in degrees [0-360].
      • windDirMax: Maximum wind direction in degrees [0-360].
      • locationID: A database identifier for this ship's record on this date. Can be used to retrieve additional details about a location if you only care about a single record.
      • returnObservations:Include textual observation associated with this record. Increased query time and size.
    • Returns Fields:
      • shipName: Exact name of ship.
      • shipType: Exact name of type of ship.
      • nationality: Exact name of nationality.
      • captainName: Exact name of captain.
      • airTemp: Air temperature in degrees celsius.
      • airPressure: Air pressure in degrees millimeters mercury.
      • gusts: Was it gusty?
      • rain: Was rain recorded?
      • snow: Was snow recorded?
      • hail: Was hail recorded?
      • thunder: Was thunder recorded?
      • fog: Was it foggy?
      • seaice: Was sea ice encountered?
      • windSpeed: Minimum wind speed in meters per second.
      • windDirMin: Minimum wind direction in degrees [0-360].
      • windDirMax: Maximum wind direction in degrees [0-360].
      • locationID: A database identifier for this ship's record on this date. Can be used to retrieve additional details about a location if you only care about a single record.
      • latitude: Latitude of matched location.
      • longitude: Longitude of matched location.
      • date: The date of this observation in format yyyy-mm-dd.

        • If the returnObservations parameter is set, these additional fields will be returned:

      • Observations: An array of observations that consists of the following objects.
        • observationID: Unique database identifier for this text observation.
        • memoText: The text of the observation as recorded in the logbook.
        • memoType: The type/content of the observation as recorded by the digitizer.
        • memoLanguage: The language that the text is written in.
    • Examples:
      • Return all observations associated with Captain James Cook's ship the Endeavour.
      • Return all of the ships of type frigate that encountered rain.
      • Return all observations for which the wind speed was between 10 and 15 mps and the wind direction was between 300 and 330 degrees.
  • nations.php Returns a list of nationalities in JSON format.
    • Accepts Parameters:
      • minDate: Time filter specifying the earliest activity of this nation. Format: yyyy-mm-dd
      • maxDate: Time filter specifying the latest voyage start date of this nation. Format: yyyy-mm-dd
    • Returns Fields:
      • Nationality: Name of assoicated nationality.
      • nationID: Database ID for this nation entity
    • Examples:
      • Return all of the nationalities in the database.
      • Return all the captains operating in or after 1845.
  • Ships.php Returns a list of ships and their associated information in JSON format.
    • Accepts Parameters:
      • shipName: Exact name of ship.
      • shipType: Exact name of type of ship.
      • nationality: Nationality associated with this ship.
      • minDate: Time filter specifying the earliest activity of this ship. Format: yyyy-mm-dd
      • maxDate: Time filter specifying the latest voyage start date from this ship. Format: yyyy-mm-dd
    • Returns Fields:
      • Type: Type of ship.
      • Information: Other information about the ship.
      • ShipName: Name of ship.
      • Notes: Any notes that we've added about this ship.
      • Nationality: Name of assoicated nationality.
      • ShipID: Database ID associated with this ship entity.
    • Examples:
      • Return all the Spanish ships.
      • Return all the captains sailing ships of type 'Frigate'.
      • Return all the ships operating between 1800 and 1805.
  • ShipTypes.php Returns a list of ship types in JSON format.
    • Accepts Parameters:
      • shipName: Exact name of ship.
      • nationality: Nationality associated with this ship.
      • minDate: Time filter specifying the earliest activity of this ship. Format: yyyy-mm-dd
      • maxDate: Time filter specifying the latest voyage start date from this ship. Format: yyyy-mm-dd
    • Returns Fields:
      • Type: Type of ship.
      • Nationality: Name of assoicated nationality.
    • Examples:
      • Return all the Spanish ship types.
      • Return all the ship types in the database.
      • Return all the ships operating between 1800 and 1805.
  • Voyages.php Returns a list of voyages and their associated information in JSON format. A voyage is a single journey between Point A and Point B made by a single ship.
    • Accepts Parameters:
      • shipName: Exact name of ship associated with this voyage.
      • shipType: Exact name of type of ship.
      • nationality: Exact name of nationality.
      • captainName: Exact name of captain.
      • rank: Exact rank name.
      • minDate: Time filter specifying the earliest activity of this captain. Format: yyyy-mm-dd
      • maxDate: Time filter specifying the latest voyage start date of this captain. Format: yyyy-mm-dd
      • fromPlace: Starting location of the voyage.
      • toPlace: Ending location of the voyage.
    • Returns Fields:
      • ShipType: Type of ship.
      • ShipName: Name of ship.
      • Captain: Name of ship captain.
      • Company: Name of company operating the voyage.
      • Nationality: Name of assoicated nationality.
      • VoyageID: Database ID associated with this ship entity.
      • FromPlace: Starting location of the voyage.
      • ToPlace: Ending location of the voyage.
      • StartDate: Starting date of the voyage in form yyyy-mm-dd
    • Examples:
      • Return all the American voyages.
      • Return all the voyages made by ships of type 'Frigate'.
      • Return all the ships operating between 1800 and 1805.
  • voyageEnds.php Returns a list of places where voyages ended (toPlace) in JSON format.
    • Accepts Parameters:
      • geocode: Only return results with valid latitude and longitude data
      • modernName: Only return results that match with a modern name.
    • Returns Fields:
      • place: Original name of location.
      • modernName: Geocoded matched name of location.
      • latitude: Latitude of matched location.
      • longitude: Longitude of matched location.
    • Examples:
      • Return all the destination ports that have lat/lng attached.
      • Return all the destination ports in the database.
      • Return all the destination ports in the database that have a modern name associated with them.
  • voyageStarts.php Returns a list of places where voyages started (fromPlace) in JSON format.
    • Accepts Parameters:
      • geocode: Only return results with valid latitude and longitude data
      • modernName: Only return results that match with a modern name.
    • Returns Fields:
      • place: Original name of location.
      • modernName: Geocoded matched name of location.
      • latitude: Latitude of matched location.
      • longitude: Longitude of matched location.
    • Examples:
      • Return all the initial ports that have lat/lng attached.
      • Return all the initial ports in the database.
      • Return all the initial ports in the database that have a modern name associated with them.
